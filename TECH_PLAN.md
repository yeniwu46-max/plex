# 个性化学习系统 - 技术方案总纲

**项目代号**: A3 | **版本**: v1.1 | **最后更新**: 2026-05-19

---

## 一、项目概述

### 1.1 项目名称
基于大模型的个性化资源生成与学习多智能体系统

### 1.2 项目目标
开发一套集学习资源管理、AI智能辅导、个性化推荐、学习效果评估于一体的智能学习平台，调动学生学习积极性，提升学习效果。

### 1.3 第一阶段目标（5月完成）
**建立基础管理系统** - 用户管理、权限管理、班级分组管理

### 1.4 后续规划
- 第二阶段：学习资源系统（资源库、个性化推荐）
- 第三阶段：学习评估系统（数据追踪、效果评估）
- 第四阶段：AI对话系统（智能辅导）
- 第五阶段：智能体协同系统（多AI协同）

## 五、核心设计决策

### 5.1 数据库设计决策

**决策**: 采用标准化关系数据库设计 + 缓存结合

| 表名 | 功能 | 关键字段 |
|------|------|---------|
| users | 用户信息 | id, username, role_id, level, points |
| roles | 角色定义 | id, name (student/teacher/admin) |
| permissions | 权限定义 | id, resource, action |
| classes | 班级信息 | id, name, teacher_id |
| achievements | 勋章定义 | id, name, rarity, condition |
| user_achievements | 用户成就 | user_id, achievement_id, unlocked_at |
| points_log | 积分日志 | id, user_id, points, reason |
| ranking_cache | 排名缓存 | class_id, user_id, rank, week |

**原因**：
- 结构清晰，易于维护
- 支持复杂查询
- 适合教学场景

### 5.2 API设计决策

**决策**: RESTful API + 统一响应格式 + JWT认证

```json
请求: GET /api/v1/users/me
响应:
{
  "code": 0,
  "message": "操作成功",
  "data": { ... }
}

错误响应:
{
  "code": 40001,
  "message": "用户不存在",
  "data": null
}
```

**原因**：
- 业界标准，易于理解
- Token无状态，适合分布式
- 前后端清晰分离

### 5.3 前端架构决策

**决策**: 游戏化UI + 激励系统可视化

- 活力配色（绿、橙、紫）
- 等级系统、勋章系统、排名榜
- 实时成就解锁动画
- 学习热力图、连续学习激励

**原因**：
- 调动学生积极性
- 参考编程竞赛平台设计
- 增强学习粘性

### 5.4 认证授权决策

**决策**: JWT Token + 基于角色的权限控制 (RBAC)

```
用户 → 查询Role → 查询Permissions → 检查权限 → 允许/拒绝
```

**三层权限体系**:
- Role: student / teacher / admin
- Permission: users:read, users:create, classes:manage, etc.
- Resource级别: 用户只能操作自己数据（除非admin）

**原因**：
- Token无状态，适合微服务
- RBAC灵活，易扩展
- 安全性高

### 8.1 用户管理模块

**功能**：
- 注册/登录（支持学生、教师、管理员）
- 用户信息增删改查
- 密码修改
- 头像上传
- 个人资料完善

**关键表**：users, roles, permissions, role_permissions

### 8.2 权限管理模块

**功能**：
- 角色定义（Student/Teacher/Admin）
- 权限分配
- 权限检查
- 资源级别的权限控制

**关键概念**：
- Role: 角色（学生、教师、管理员）
- Permission: 权限（users:read, users:create等）
- Resource: 资源（users, classes等）
- Action: 操作（read, create, update, delete）

### 8.3 班级管理模块

**功能**：
- 班级创建/编辑
- 学生分配（单个/批量导入）
- 班级成员管理
- 班级统计信息

**关键表**：classes, 用户-班级关系

### 8.4 激励系统模块

**功能**：
- 用户等级（Lv1-Lv5）
- 积分系统（任务完成、每日签到等）
- 勋章系统（19种勋章）

**关键表**：achievements, user_achievements, points_log, ranking_cache

### 8.5 试炼竞技场模块

**功能**：
- 4大试炼模式（AI对抗、星域竞技、自我突破、深渊试炼）

- 积分获取与资源奖励
- 用户等级进度展示
- 资源系统管理

**关键组件**：
- `TrialArenaView.vue`：完整页面布局（侧边栏+地图+右侧面板）

- `GET /api/v1/user/resources` - 用户资源余额

## 十二、术语表

| 术语 | 定义 |
|------|------|
| RBAC | 基于角色的权限控制 (Role-Based Access Control) |
| JWT | JSON Web Token，无状态认证 |
| ORM | 对象关系映射 (Object-Relational Mapping) |
| RESTful | 表现层状态转移 (Representational State Transfer) |
| 激励系统 | 通过等级、勋章、排名等机制调动用户积极性 |
| 多智能体系统 | 多个AI模型协同工作 |
