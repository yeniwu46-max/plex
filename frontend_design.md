# 前端设计方案 - 个性化学习系统

## 一、设计风格指南

### 1. 整体风格定位
- **风格**：现代简洁 + 教育友好
- **主题**：Light Mode（默认）+ Dark Mode（可选）
- **组件库**：Naive UI（提供现成UI组件，开箱即用）
- **布局**：左侧导航栏 + 顶部用户栏 + 主内容区

### 2. 色彩系统
| 角色 | 主色 | 辅助色 | 说明 |
|------|------|--------|------|
| 学生端 | 蓝色（#3B82F6） | 青色/绿色 | 学习、探索的含义 |
| 教师端 | 紫色（#8B5CF6） | 橙色 | 管理、指导的含义 |
| 管理员端 | 红色（#EF4444） | 黄色 | 权限、控制的含义 |

### 3. 字体排版
- **标题**：20-28px，字重600-700
- **正文**：14px，字重400，行高1.6
- **标签/按钮**：12-14px，字重500

## 三、页面设计清单

### A. 公共页面（所有用户）

#### 1. **登录页** (`/login`)
**内容**：
- 顶部：系统Logo + 应用标题
- 中央：登录表单
  - 账号/邮箱输入框
  - 密码输入框
  - 记住我 (checkbox)
- 底部：忘记密码 + 需要帮助
- 背景：渐变色或教育主题图

**布局**：全屏居中，宽度500px

---

#### 2. **忘记密码页** (`/forgot-password`)
**内容**：
- 账号输入框 → 验证码发送
- 邮箱接收验证码
- 新密码设置
- 返回登录链接

---

#### 3. **个人中心** (`/profile`)
**内容**：
- 左侧：用户头像 + 基本信息
- 右侧：可编辑的用户信息表单
  - 姓名、邮箱、电话
  - 所在班级/学科
  - 头像上传
  - 密码修改
- 底部：保存 / 取消按钮

---

### B. 学生端页面

#### 1. **学生首页/仪表板** (`/student/dashboard`)
**内容**：
- 顶部欢迎区：学生姓名 + 今日任务数
- 左上卡片：学习进度统计
  - 已完成课程数
  - 总学习时长
  - 本周学习天数
- 中间：学习路径卡片（3个）
  - 课程名称 + 进度条
  - 快速进入按钮
- 右上卡片：推荐资源（3条）
  - 资源类型 + 标题
  - 推荐理由

**配色**：蓝色主题

---

#### 2. **学生课程列表** (`/student/courses`)
**内容**：
- 上方：搜索框 + 分类筛选（全部/进行中/已完成）
- 课程卡片网格（每行3个）
  - 课程缩略图/Banner
  - 课程名称 + 教师名
  - 进度条
  - 学习时长 / 作业数
  - 进入按钮

**布局**：卡片网格布局，响应式

---

#### 3. **学生个人资料与学习画像** (`/student/profile`)
**内容**：
- 基本信息卡片
- 学习风格标签（可视化展示）
- 学习数据统计
  - 总学习时长
  - 完成度
  - 平均成绩
- 修改个人信息按钮

---

### C. 教师端页面

#### 1. **教师首页** (`/teacher/dashboard`)
**内容**：
- 顶部：班级切换下拉框
- 统计卡片（4个）

  - 学生名 + 学号

  - 学习进度

#### 2. **班级管理** (`/teacher/classes`)

  - 学年

- 学习数据统计

  - API响应时间

#### 2. **用户管理** (`/admin/users`)

  - 用户ID / 姓名 / 邮箱
  - 角色 / 创建时间

  - 操作（编辑/冻结/删除）
- 创建用户按钮 → 表单弹窗
- 批量操作（导入/导出）

**内容**：
- 角色列表
  - 角色名 / 描述 / 人数
  - 编辑权限按钮
- 权限编辑弹窗
  - 权限项树形结构
  - 勾选框（授予/取消权限）
  - 保存按钮

---

#### 4. **班级分组管理** (`/admin/classes`)
**内容**：

  - 班级名 / 年级 / 人数
  - 创建时间 / 操作

- 创建班级表单
- 学生分配功能

## 四、关键组件设计

### 1. **导航栏组件**
```
学生端导航：
├── 首页
├── 我的课程
├── 学习资源（预留）
├── 学习报告（预留）

教师端导航：
├── 首页
├── 班级管理
├── 学生管理

├── 首页
├── 用户管理
```

### 2. **顶部栏组件**
- 左：Logo + 应用名
- 中：面包屑导航

- 支持排序、筛选、分页
- 操作列固定

### 4. **表单组件**
- 使用Naive UI Form
- 验证反馈即时显示
- 必填字段标红

### 5. **模态框（Modal）**
- 创建/编辑用户
- 权限分配
- 确认删除

### 6. **卡片组件**
- 学习进度卡片
- 统计数据卡片
- 课程卡片

## 五、交互流程

### 登录流程
```
登录页 → 验证账号密码 → 后端验证 → 生成Token
→ 存储到localStorage → 跳转到对应首页
```

### 权限校验流程
```
访问页面 → 检查Token → 检查用户角色 
→ 调用权限接口 → 无权限显示403 → 有权限加载页面
```

## 九、核心交互特性

### 1. 即时反馈
- 表单验证错误红色标记
- 操作成功/失败toast提示
- 加载状态spinner显示

### 2. 面包屑导航
- 方便用户定位当前位置
- 支持快速返回上级

### 3. 权限隐藏
- 无权限菜单项直接隐藏
- 无权限页面显示403

### 4. 快捷操作
- 表格行的快速操作按钮
- 右键菜单支持（预留）

## 十、设计规范总结

| 方面 | 规范 |
|------|------|
| 字体 | 系统默认字体 (Helvetica/Microsoft YaHei) |
| 间距 | 8px基础单位 |
| 圆角 | 按钮4px，卡片8px |
| 阴影 | 轻微阴影(0 2px 8px rgba(0,0,0,0.1)) |
| 动画 | 300ms过渡时间 |
| 按钮 | 高度40px，内边距12px |
