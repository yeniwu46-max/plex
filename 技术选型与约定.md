## 1. 前端定案

| 类别 | 选型 | 说明 |
|------|------|------|
| 框架 | **Vue 3** | 正式工程唯一目标前端框架 |
| 构建 | **Vite** | 开发与打包 |
| UI | **Naive UI** | 与 [TECH_PLAN.md](TECH_PLAN.md) 总纲一致 |
| 状态 | **Pinia** | 全局状态 |
| 请求 | **Axios** | HTTP；Token 等放在拦截器统一处理 |
| 动效（可选） | **motion-v** | 声明式属性动画、与 Vue 模板配合；见下文 **1.2** |

### 1.1 与 AI 生成代码的关系

- 常见 AI 工具（如 Claude）训练语料偏 **React**，生成 React 往往更顺手。
- **团队仍选定 Vue**：本仓库后续新增页面与模块，以 **Vue 3** 为唯一目标栈，避免长期双栈维护。
- 若已有 **React** 原型（例如登录页、动效 Demo），其角色定为 **视觉与交互参考**；纳入本仓库的正式前端时，采用 **Vue 重写或迁移**，不以「长期并存 React + Vue」为目标。

### 1.2 动效：motion-v

需要**组件级位移 / 过渡 / 交互动效**时，优先在 Vue 内用 **motion-v**（与 Motion 生态一致，避免把 React 版 `framer-motion` 写法直接搬进 SFC）。

安装：

```bash
npm install motion-v
```

**模板里绑定对象**须用 Vue 的 `v-bind` 对象字面量，**不要**写成 JSX 的 `{{ }}`（在 SFC 中 `{{ }}` 是文本插值，不是对象）。

```vue
<script setup lang="ts">
import { motion } from 'motion-v'
</script>

<template>
  <motion.div :animate="{ x: 100 }" />
</template>
```

更复杂的 `animate`、`transition`、`whileHover` 等以 [motion 官方文档](https://motion.dev/) 与 `motion-v` 包说明为准；页面级滚动入场等仍可与 **Animate.css / AOS** 并存，见 [frontend_design_v2.md](frontend_design_v2.md) 第九章。

## 2. 后端定案

| 类别 | 选型 | 说明 |
|------|------|------|
| 框架 | **Flask** | Web 与 API |
| ORM | **SQLAlchemy**（Flask-SQLAlchemy） | 表与模型映射，减少手写 SQL；表结构变更通过迁移与模型同步 |
| 生产数据库 | **MySQL** | 主库定案 |
| 本地开发 | **SQLite**（可选） | 与 [backend/QUICKSTART.md](backend/QUICKSTART.md) 一致时可不装 MySQL，仅用于开发自测 |
